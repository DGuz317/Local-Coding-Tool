# RepoLens MCP v0.5.0 - 2026-07-02

## Summary

RepoLens v0.5.0 makes Assistant Preflight the recommended first step before broad repository reads. The release adds deterministic task orientation, focus hints, candidate command metadata, local savings estimates, and artifact safety checks while preserving the local-first, read-only, no-source-disclosure boundary.

## Added

- Assistant Preflight CLI: `repolens preflight /path/to/repo "Describe your task"`.
- MCP `assistant_preflight` tool for bounded assistant orientation before file exploration.
- Parser backend contract and experimental parser hash boundaries.
- Resolver evidence taxonomy, relationship candidates, and graph-quality warnings.
- High-level graph-store seam over the SQLite artifact contract.
- Deterministic focus hints, budget metadata, stale graph handling, and broad-task/no-match behavior.
- Local `evaluate-context` savings metrics and a v0.5 dogfood evaluation pack.
- `audit-artifacts` safety audit for generated artifacts and representative assistant-facing output.
- GitHub Actions workflows for fast CI, release gate checks, and Docker smoke coverage.

## Changed

- README and assistant documentation now direct assistants to call preflight before broad file reads.
- Candidate verification commands remain discovered metadata only and are explicitly marked `run: false`.
- Context Pack and preflight output continue to prefer compact metadata over source content.
- Package/workspace evidence preserves uncertainty through candidates, warnings, unresolved states, and unsupported states.

## Security And Privacy

- Artifact audit checks generated `.repolens/` files and representative assistant-facing output for source snippet leakage, absolute host paths, raw secrets, raw Agent Guidance mirroring, oversized artifacts, and contract violations.
- Normal indexing and MCP usage remain local-first and do not require hosted services, telemetry, embeddings, browser UI, or runtime network calls.

## MCP Contract

- Added `assistant_preflight` as the bounded first-call workflow for assistant task orientation.
- Preflight responses include freshness, first-read files, likely tests, candidate commands, focus hints, warnings, confidence, limits, truncation, and budget metadata.
- MCP tools remain read-only.

## Known Limitations

- See `docs/known-limitations.md` for release-relevant limitations.
- PyPI publishing and Docker registry publishing remain deferred.
- Deep semantic call graphs, embeddings, hosted search, HTTP serving, watch mode, and write-capable MCP tools remain out of scope.
