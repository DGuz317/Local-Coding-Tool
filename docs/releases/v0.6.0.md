# RepoLens MCP v0.6.0 - 2026-07-06

RepoLens v0.6.0 improves JavaScript and TypeScript assistant orientation while preserving the local-first, read-only, no-source-disclosure boundary. The release adds parser-backed JS/TS structure when Tree-sitter support is available, richer resolver outcome metadata, source-free Call Chain Facts, Framework Route Hints, and release-gate coverage for the new facts.

## Highlights

- Tree-sitter JS/TS is the default parser backend when the parser and grammar packages are available.
- Legacy bounded scanner fallback remains available and emits parser-backend warnings when Tree-sitter support is unavailable.
- Stable JS/TS parser facts include source-free module, import, export, top-level symbol, parser provenance, and line-range metadata.
- Call Chain Facts provide structural metadata for assistant orientation without claiming runtime reachability.
- Resolver outcomes for imports, exports, aliases, and workspace package relationships preserve uncertainty with candidates, unresolved statuses, Relationship Candidates, and Graph Quality Warnings.
- Framework Route Hints provide bounded local hints for framework-shaped repositories, with the first fixture covering Next.js App Router paths.
- Assistant Preflight and Impact Analysis can use the richer JS/TS metadata to improve first-read ranking and impact hints.
- Artifact audit coverage includes v0.6 facts and checks that assistant-facing output remains source-free and path-safe.

## Assistant Workflow Impact

Assistants should still call `assistant_preflight` before broad repository exploration. v0.6 makes that first call more useful for JS/TS tasks by surfacing parser and resolver evidence, route hints, call-chain metadata, likely tests, warnings, and candidate verification commands as compact metadata.

Treat these facts as orientation only:

- Call Chain Facts are not runtime proof or a semantic call graph.
- Framework Route Hints are not framework emulation or runtime route proof.
- Resolver outcomes are not definitive when RepoLens reports ambiguity, unsupported cases, candidates, or unresolved statuses.
- Candidate verification commands are found but not run.

## Safety And Privacy

v0.6 does not add hosted services, telemetry, embeddings, browser UI, write-capable MCP tools, package registry lookups, package-manager execution, compiler execution, bundler execution, framework execution, or test execution during indexing or assistant context generation.

Generated artifacts and assistant-facing output must not contain source snippets, code bodies, function signatures, full import lines, raw comments, raw Agent Guidance text, raw config values, raw secrets, or absolute host paths. `.repolens/` remains private local metadata and should not be committed or published without review.

## Known Limitations

- Tree-sitter JS/TS parser-backed facts require local parser and grammar packages. Without them, RepoLens falls back to the legacy bounded scanner with warnings.
- JS/TS resolution remains local-evidence-limited. Unsupported aliases, complex package entrypoints, conditional exports, generated entrypoints, and framework conventions may remain unresolved or candidates.
- Next.js App Router route hints are deterministic hints only. RepoLens does not run Next.js or inspect build output.
- Call Chain Facts are shallow parser-derived structural metadata, not runtime reachability, data-flow, or framework lifecycle analysis.
- Parser timing and file-count evidence are bounded local fixture evidence only. They do not justify parse caches, worker pools, indexing parallelism, telemetry, package-manager execution, compiler execution, or framework execution.

See `docs/known-limitations.md` for the full release limitation list.

The release gate is:

```bash
uv run pytest
uv run ruff check .
uv run ruff format --check .
uv run mypy src/repolens
uv run repolens evaluate-context --json
uv run repolens audit-artifacts . --json
uv build --out-dir /tmp/repolens-dist --clear
```

This release note does not publish to PyPI, push a Docker image, or start any hosted service.
