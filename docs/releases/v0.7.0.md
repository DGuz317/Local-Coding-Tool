# RepoLens MCP v0.7.0 - 2026-07-09

RepoLens v0.7.0 adds the Python Semantic Analysis Prototype while preserving the local-first, read-only, no-source-disclosure boundary. The release introduces experimental, source-free Python control-flow and lexical binding metadata stored separately from the stable graph so maintainers can inspect and evaluate semantic facts without changing default assistant-facing behavior.

## Highlights

- Experimental semantic facts are stored outside the stable graph, with `.repolens/semantic.sqlite` as the indexed semantic artifact.
- `repolens semantic-inspect` reads indexed semantic artifacts by default and reports missing, stale, or incompatible artifacts instead of silently parsing live source.
- `semantic-inspect --from-source` provides explicit, non-persistent debug output for local inspection.
- Python function-level control-flow facts cover supported entry, branch, loop, return, raise, exit, unsupported, and uncertain paths.
- Python lexical binding facts cover local definitions, parameters, imports, assignments, references, unresolved names, free-variable candidates, shadowing, `global`, and `nonlocal` declarations where deterministic AST evidence exists.
- Structured semantic warnings preserve uncertainty for unsupported, dynamic, ambiguous, or intentionally skipped constructs.
- Deterministic semantic fixture evaluation covers CFG, lexical binding, warning, no-disclosure, stable identity, and artifact-audit evidence.
- Optional Context Pack semantic hints are available only behind explicit `include_experimental_semantic_hints` opt-in.
- Default MCP, Assistant Preflight, Context Pack, Canonical Graph Hash, and stable graph behavior do not include semantic facts by default.

## Assistant Workflow Impact

Assistants should still call `assistant_preflight` before broad repository exploration. v0.7 adds an explicit semantic inspection path for Python repositories, but semantic facts remain experimental candidate metadata.

Treat these facts as orientation only:

- Control-flow facts are not runtime reachability proof.
- Lexical binding facts do not prove runtime values or dynamic Python behavior.
- Semantic warnings are evidence of unsupported or uncertain constructs, not defects.
- Optional semantic hints are source-free and opt-in only.
- Candidate verification commands are found but not run.

## Safety And Privacy

v0.7 does not add hosted services, telemetry, embeddings, browser UI, write-capable MCP tools, package registry lookups, package-manager execution, compiler execution, framework execution, runtime Python execution, or test execution during indexing or assistant context generation.

Semantic artifacts and assistant-facing output must not contain source snippets, code bodies, function signatures, raw condition text, raw expression text, raw values, raw comments, raw docstrings, raw string literals, raw secrets, raw Agent Guidance text, absolute host paths, or AI prose summaries. `.repolens/` remains private local metadata and should not be committed or published without review.

## Known Limitations

- v0.7 semantic analysis is Python-first and experimental.
- Semantic facts are not part of the stable graph contract.
- Semantic facts are excluded from Canonical Graph Hash, default Context Pack IDs, stable graph validation, default MCP output, default Assistant Preflight output, and default Context Pack output.
- `--from-source` inspection is explicit debug output and does not persist artifacts.
- v0.7 does not include data-flow analysis, taint analysis, reaching definitions beyond bounded lexical binding facts, whole-program semantic analysis, type inference, JS/TS semantic analysis, or runtime Python behavior proof.

See `docs/known-limitations.md` for the full release limitation list.

uv run mypy src/repolens
uv run repolens evaluate-context --json
uv run repolens evaluate-semantics --export-debug-jsonl --json
uv run repolens audit-artifacts . --json
uv build --out-dir /tmp/repolens-dist --clear
