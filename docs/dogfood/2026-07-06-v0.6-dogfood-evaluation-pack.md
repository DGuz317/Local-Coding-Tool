## Findings

- The v0.6 release gate now covers call chains, alias ambiguity, re-export behavior, workspace package imports, route hints, stale graph behavior, and no-source-disclosure negatives.
- Dogfood evidence remains fixture-derived and compact: paths, names, bounded line ranges, warning codes, and metadata only.
- Local savings metrics remain fixture-derived estimates and are not telemetry, exact model-token claims, or productivity claims.
- Bounded local parser/index timing and eligible file counts are recorded as evaluation evidence only.
- Performance evidence documents current limitations; it does not add parse cache, worker pools, indexing parallelism, or runtime package-manager/compiler/framework execution.

## Known Limitations

- The pack uses committed distilled fixtures instead of vendoring large external repositories.
- JS/TS resolution remains local-evidence-limited; unsupported aliases, package entrypoints, conditional exports, and framework conventions remain unresolved or candidates.
- Next.js App Router route hints are metadata hints only. RepoLens does not execute Next.js, inspect runtime routes, or emulate framework loaders.
- Call-chain facts are shallow parser facts with method names, receiver shape, and bounded line ranges; they are not a semantic call graph.
- Re-export facts describe local export metadata, not runtime package export-map behavior.
