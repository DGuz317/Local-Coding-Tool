# RepoLens MCP v0.8 Release Checklist

This checklist gates release readiness. It does not publish to PyPI, Docker registries, or hosted services.

## Human Checkpoint

A human maintainer must confirm before final release publication:

- README quickstart, safety overview, and assistant guidance are accurate.
- `docs/assistant-usage-guide.md` and `docs/mcp-tool-examples.md` match the current MCP contract.
- `docs/security-and-artifact-privacy.md` accurately describes scanner, redaction, artifact, and MCP safety behavior.
- `docs/troubleshooting.md` covers the known operational failure modes.
- `docs/known-limitations.md` reflects dogfooding outcomes and does not hide release-blocking failures.
- `docs/releases/v0.8.0.md` describes the release without unsupported publishing or external-provider quality claims.
- `docs/ai-proposals.md` matches disabled-default behavior, supported proposal kinds, persistence, Artifact Safety Audit, and deferred Active Workflow boundaries.
- `docs/dogfood/2026-07-10-v0.8-ai-proposal-layer.md` records proposal usefulness, overclaim review, and unavailable external-provider dogfood.
- Release notes use the changelog template and avoid unsupported publishing claims.
- PyPI publishing automation remains out of scope.
- Docker registry publishing automation remains out of scope.

## Automated Verification Gate

Run from the repository root:

```bash
uv run pytest
uv run ruff check .
uv run ruff format --check .
uv run mypy src/repolens
uv run repolens evaluate-context --json
uv run repolens index . --json
uv run repolens audit-artifacts . --include-ai-proposals --json
uv build --out-dir /tmp/repolens-dist --clear
```

## Build And Install Smoke

```bash
uv build
uv tool install --force dist/*.whl
repolens --help
repolens status .
uv tool uninstall repolens
```

## CLI Smoke

```bash
uv run repolens --help
uv run repolens status .
uv run repolens index .
uv run repolens update .
uv run repolens report .
uv run repolens preflight . "Check release readiness" --json
```

Do not commit `.repolens/` generated by smoke tests.

## AI Proposal Smoke

On an indexed temporary copy of the repository:

- Confirm a request without `--enable-ai` returns `disabled` without a provider call, network access, or file write.
- Request Context Pack Summary, Architecture Explanation, and Patch Plan output with the local `test` provider.
- Repeat equivalent packed input and confirm a stable `input_digest`.
- Confirm Context Pack ID and Canonical Graph Hash are unchanged after proposal generation.
- Save one proposal explicitly and run `repolens audit-artifacts <repo> --include-ai-proposals --json`.
- Review output for useful evidence refs and plausible targets/tests, and record overclaiming or implied command, write, mutation, patch, or remote-post behavior.

External-provider quality dogfood is unavailable in v0.8 because no external provider is supported. Use deterministic test-provider contract evidence without presenting it as external-model quality.

## Docker Smoke

```bash
docker build -t repolens:latest .
docker run --rm --network none --user "$(id -u):$(id -g)" -v "$PWD:/workspace" repolens:latest index /workspace
docker run --rm --network none --user "$(id -u):$(id -g)" -v "$PWD:/workspace" repolens:latest status /workspace
```

For Docker MCP, start the stdio server through an MCP client or with stdin attached:

```bash
docker run --rm -i --network none --user "$(id -u):$(id -g)" -v "$PWD:/workspace" repolens:latest mcp /workspace
```

## MCP Smoke

Using an MCP client, confirm:

- The client can list RepoLens tools.
- `graph_status` returns a standard envelope.
- `repo_summary` returns repository metadata.
- `assistant_preflight` returns bounded task orientation before broad file reads.
- `get_task_context` returns a bounded Context Pack for a real task.
- `impact_analysis` returns grouped context for a known file.
- `search_text` returns capped previews and not whole files.
- Missing or stale artifacts produce actionable envelope errors or warnings.

## Dogfooding Evidence

Confirm dogfooding reports cover:

- RepoLens on itself.
- At least one representative Python repository.
- At least one representative JS/TS repository or distilled fixture when no suitable local repo is available.
- At least one mixed docs/config repository or distilled fixture.
- Regression fixtures for actionable findings where practical.

## Scope Guard

Do not add or require these for v0.8 release readiness:

- PyPI publishing automation.
- Docker registry publishing automation.
- Hosted release channels.
- Runtime network calls during normal indexing or MCP serving.
- Browser UI, HTTP API, telemetry, embeddings, or write-capable MCP tools.
- External AI providers, hidden provider fallback, raw-source prompting, remote review posting, or Active Workflow behavior.
