# RepoLens MCP v0.8.0 - 2026-07-10

RepoLens v0.8.0 adds a narrow, optional AI Proposal Layer over bounded RepoLens metadata. AI Proposals are labeled interpretations outside the Deterministic Graph Foundation; AI remains disabled by default and normal indexing, graph queries, Context Packs, Assistant Preflight, and MCP behavior do not require a provider.

## Highlights

- One read-only `create-ai-proposal` CLI and MCP surface with structured disabled, unavailable, and available results.
- Explicit provider and model selection with provenance, input packer version, redaction policy version, input digest, graph schema version, and Canonical Graph Hash context where applicable.
- Context Pack Summary, Architecture Explanation, and Patch Plan proposal kinds.
- Deterministic, redacted, size-bounded metadata input instead of whole source files or raw prompt persistence.
- Ephemeral output by default; explicit `--save` persists a bounded artifact under `.repolens/ai-proposals/` and runs Artifact Safety Audit.
- The local deterministic `test` provider exercises the contract without network access or hidden provider fallback.

## Trust And Workflow Boundaries

AI Proposals do not change deterministic graph facts, Canonical Graph Hash, Context Pack IDs or ranking, Task Matching, resolver behavior, Package Ownership, or graph traversal. They cannot be promoted directly into graph facts.

Patch Plan Proposals are planning metadata only. v0.8 does not apply patches, write project files, execute commands, mutate branches, or post remote comments. Candidate Verification Commands remain marked not run. Active Workflow and richer proposal types remain deferred beyond v0.8.

## Known Limitations

- The only supported provider is the local deterministic `test` provider; v0.8 does not establish external-model quality.
- Proposal quality is bounded by indexed metadata, Context Pack coverage, graph freshness, ambiguity, and truncation.
- Dogfooding found that a docs-focused Patch Plan could identify the core implementation file and related tests while omitting the supplied documentation target and Candidate Verification Commands. Proposed targets are orientation, not guaranteed edit files.
- AI input excludes whole source by default, so proposals cannot replace scoped source inspection.
- Saved AI Proposal audit is a bounded artifact-contract check, not a general secret scanner.

See `docs/known-limitations.md` and `docs/dogfood/2026-07-10-v0.8-ai-proposal-layer.md` for full limitations and dogfood evidence.

## Verification

Release-readiness evidence is recorded in `docs/release-readiness.md`. The release gate includes:

```bash
uv run pytest
uv run ruff check .
uv run ruff format --check .
uv run mypy src/repolens
uv run repolens evaluate-context --json
uv run repolens audit-artifacts /tmp/repolens-v08-dogfood --include-ai-proposals --json
uv build --out-dir /tmp/repolens-dist --clear
```

External-provider dogfood could not be run because v0.8 supports no external provider. Replacement evidence is the deterministic test-provider dogfood, repeated-input digest check, schema and safety contract tests, stable Context Pack ID and Canonical Graph Hash checks, and saved-proposal Artifact Safety Audit.

This release note does not claim that a package was published, an image was pushed, commands were executed by AI Proposal output, or any hosted service was started.
