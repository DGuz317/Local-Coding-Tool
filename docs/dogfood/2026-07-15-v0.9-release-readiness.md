## Performance Evidence

- Selective Update: 1.206 seconds;
- full rebuild: 0.803 seconds;
- relative factor: 0.666, where values above 1.0 mean Selective Update was faster.

The benchmark and dogfood runs do not show a consistent Selective Update speed advantage in this environment. No parse cache, worker pool, or parallel indexing complexity was added. The evidence supports keeping the current simple implementation and treating optimization as follow-up work only when representative adoption is blocked.

## Setup Friction And Boundaries

The supported first-use path is install, register `repolens mcp /absolute/path/to/repo`, then call `assistant_preflight`. No RepoLens project configuration or separate indexing command is required. Optional focus hints and budget controls can narrow output; experimental semantic hints and AI Proposals remain explicit opt-ins under their documented contracts.

RepoLens remains local-first and has no telemetry. MCP tools remain read-only. Indexing and first use do not execute repository package managers, compilers, bundlers, frameworks, tests, deploy commands, or publish commands. Package and container publishing remain outside this release gate.
