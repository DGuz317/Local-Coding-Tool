# RepoLens MCP v0.9.0 - 2026-07-15

## Summary

RepoLens v0.9.0 makes Assistant Preflight the primary zero-configuration, agent-native workflow. After one local installation and MCP client registration, the first preflight call discovers the repository and initializes missing graph state; later calls refresh stale state before returning bounded orientation metadata.

## Added

- Automatic missing-graph initialization through MCP `assistant_preflight`.
- Automatic stale-graph refresh through the same preflight workflow.
- Zero-configuration repository structure detection coverage for supported Python and JavaScript/TypeScript repositories.
- Progressive-disclosure improvements for First-Read Files, Related Tests, risks, warnings, focus hints, and deterministic expansion handles.
- Local context relevance and exploration-savings evaluation against a deterministic lexical baseline.
- Packaged first-use MCP smoke coverage for clean installation, initialization, and refresh.

## Changed

- Installation plus MCP client registration is now the recommended setup boundary; a separate indexing command is not required before the first Assistant Preflight call.
- Context hygiene excludes or deprioritizes generated, vendored, cached, binary, unsupported, duplicate, and weakly relevant material while preserving uncertainty.
- User documentation is consolidated around Assistant Preflight. Explicit `index`, `update`, `status`, and artifact-audit commands remain available for diagnostics and release verification.
- Context Pack budgets, ranking, warnings, and safety checks provide clearer bounded-orientation metadata.

## Security And Privacy

- MCP tools remain read-only.
- Normal indexing and preflight remain local-first and do not add telemetry, hidden network access, package-manager/compiler/bundler/framework execution, or repository command execution.
- Assistant-facing output remains bounded metadata with repo-relative POSIX paths and No Whole-Source Disclosure.
- Candidate Verification Commands remain marked not run.
- Artifact Safety Audit explicitly covers redaction, bounded output, repo-relative paths, deterministic ordering, source-disclosure boundaries, and not-run commands.

## MCP Contract

- `assistant_preflight` is the recommended first call before broad repository reads.
- Missing or stale graph state is handled automatically and reported through freshness and warning metadata.
- Focus hints and budget controls remain optional deterministic inputs.
- Experimental semantic hints and AI Proposals remain explicit opt-ins and do not alter default graph truth.

## Known Limitations

Sparse Task Matching evidence can produce no First-Read Files even when relevant files exist; callers should retry with a repo-relative focus hint and inspect warnings. Local dogfood also found that Selective Update is not consistently faster than full rebuild, and local context evaluation does not support an exact or universal token-saving claim.

See `docs/known-limitations.md` and `docs/dogfood/2026-07-15-v0.9-release-readiness.md`.
