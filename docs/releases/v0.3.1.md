# RepoLens v0.3.1 Release Notes

RepoLens v0.3.1 is an artifact usability patch for repositories where `.repolens/graph-index.md` could grow too large to open comfortably.

## Bug Fixed

Earlier `graph-index.md` output behaved too much like a Markdown dump of graph facts. On larger repositories, that made an assistant-facing artifact expensive to load and inconsistent with RepoLens' context-budget goal.

## New Artifact Policy

The default `.repolens/graph-index.md` is now a bounded navigation landing page. It is capped by deterministic total-character and section-row budgets, and it should be used to choose targeted follow-up inspection rather than to read every graph fact.

SQLite remains the complete graph source of truth. The bounded Markdown index may omit lower-priority rows, but those graph facts remain available in `.repolens/graph.sqlite` and structured exports.

## Truncation Metadata

When rows are omitted, `.repolens/graph-status.json` reports `exports.graph_index.truncated` and section metadata including `shown`, `total`, and `reason`. The metadata explains the artifact budget boundary; it is not a claim that omitted rows are irrelevant.

## Retrieving Omitted Facts

Use bounded graph metadata search for omitted rows instead of loading or generating large Markdown:

```bash
uv run repolens search-graph . auth --kind symbol --limit 20 --json
uv run repolens search-graph . login --kind file --limit 20 --json
uv run repolens search-graph . test --kind command --limit 20 --json
```

For complete inspection, query `.repolens/graph.sqlite` directly or inspect `.repolens/graph.json` with targeted filters.

The full or sharded Markdown export path is optional release-follow-up behavior and is not part of the default `graph-index.md` path. If enabled later, it must preserve the same no whole-source disclosure rules.

## Safety Boundary

Generated Markdown should not mirror full source files, source snippets, code bodies, raw comments, raw guidance text, secret-like task text, or absolute host paths. RepoLens artifacts remain orientation metadata, not source preview bundles.
